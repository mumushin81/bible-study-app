# HebrewIcon 컴포넌트 변경 사항

## 주요 개선 사항

### 1. URL 검증 로직 확장
- UUID 형식의 파일명 패턴 지원 강화
- 언더스코어로 구분된 UUID 형식 허용
- 기존 Supabase 이미지 URL 호환성 개선

#### 변경 전 패턴
```regex
/^https:\/\/[a-zA-Z0-9-]+\.supabase\.co\/storage\/v1\/object\/public\/hebrew-icons\/icons\/word_[a-f0-9]{32,}\.jpg$/
```

#### 변경 후 패턴
```regex
/^https:\/\/[a-zA-Z0-9-]+\.supabase\.co\/storage\/v1\/object\/public\/hebrew-icons\/icons\/word_(?:[a-f0-9]{32}|[a-f0-9]{8}_[a-f0-9]{4}_[a-f0-9]{4}_[a-f0-9]{4}_[a-f0-9]{12})\.jpg$/
```

### 2. 이미지 에러 상태 관리 개선
- `imageError` 상태 재설정 로직 추가
- 새로운 유효한 URL이 들어올 때 에러 상태 초기화
- 불필요한 이미지 로딩 실패 방지

### 3. 불필요한 코드 제거
- 사용하지 않는 Supabase 클라이언트 초기화 제거
- 불필요한 자동 생성 옵션 제거
- 컴포넌트 복잡성 감소

## 개선 효과
- 더 유연한 이미지 URL 처리
- 이미지 로딩 실패 시 복구 메커니즘 추가
- 번들 크기 최적화
- 코드 가독성 향상

## 주의사항
- 기존 이미지 URL 구조 완전히 지원
- 에러 상태 관리 로직 변경으로 이미지 로딩 안정성 개선
