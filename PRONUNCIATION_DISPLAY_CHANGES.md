# 발음 표시 변경 노트

## 주요 변경 내용
- 복잡한 음절 생성 로직 완전 제거
- 직관적인 영어 발음 변환 도입
- 모든 발음 표시 로직 단순화

## 변환 규칙
- 모음 변환 맵:
  - 'a' → 'ah'
  - 'e' → 'eh'
  - 'i' → 'ee'
  - 'o' → 'oh'
  - 'u' → 'oo'

## 적용 위치
- 어근 분석 섹션
- 히브리어 문자별 발음 표시

## 변경 목적
- 코드 복잡성 제거
- 발음 표시의 직관성 향상
- 유지보수성 개선

## 예시
- 이전: 복잡한 음절 분리 알고리즘
- 현재: 간단한 문자 대체 방식

## 주의사항
- 발음의 정확성보다는 직관성 중시
- 학습자가 쉽게 이해할 수 있는 형태로 변환
